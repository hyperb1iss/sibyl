01 / The verdict

Sibyl is closer than the roadmap admits.

The core is no longer a memory experiment. It is a credible multi-tenant context runtime with provenance, tasks, synthesis packets, usage feedback, team scopes, Open Knowledge Format export, and unusually honest evaluation machinery. The next leap is not another retrieval trick. It is turning that runtime into a trustworthy home for an entire life.

Strategic conclusion
Build the sovereign context operating system.

A provenance-complete evidence fabric that turns raw experience and agent trajectories into precise, permissioned, replayable context and reusable competence. Not a chatbot. Not a vector database. Not a note app. Not an agent framework.

Scale trap 01

The import parent row grows forever.

A source import currently accumulates raw IDs, source IDs, dedupe keys, skipped records, and source-to-memory mappings on one parent. At hundreds of thousands of items, resume repeatedly reconstructs and serializes a giant mutable object.

Scale trap 02

Email batches revisit the archive.

MBOX batches reopen and materialize keys; Maildir batches rebuild and sort the listing. Resume is ordinal-based rather than byte-indexed, turning a large archive into repeated whole-corpus work.

Truth trap 03

“Raw” mail is extracted text.

A sovereign archive must preserve exact RFC/MBOX bytes, MIME structure, labels, headers, and attachment hashes. Parsed text is a useful projection. It is not raw evidence.

Subtract before expanding

Make one obvious path through the power already here.

An internal audit of 1,732 local agent transcripts found ritual-heavy read verbs, seven importance keys, four project-scope expressions, narrow use of a 33-kind ontology, and correction machinery separated from ordinary capture. Unify the authoring and recall surfaces, collapse duplicate fields and scope resolution, and advertise a smaller coherent vocabulary without deleting the richer backend model. Transcript counts are host-conditioned evidence, not a referendum on capability.

03 / The field

The systems surveyed here each own a piece. None spans this whole.

The frontier is fragmenting into coding provenance, agent memory, temporal graphs, local capture, portable data, and platform-managed context. Sibyl’s opportunity is to compose those truths into one governed life substrate without pretending one index solves every problem.

Coding provenance

Entire

Git-linked checkpoints preserve prompts, transcripts, tool calls, file changes, tokens, and attribution. They support review and search; resume and rewind are available where the agent integration supports them.

Steal the atom: generalized evidence-bound checkpoints. Ingest Entire; do not turn Git into the universal store for human life.

Agent memory

Letta · Mem0 · LangMem

Memory blocks, archival tiers, file search, extraction, consolidation, and agent state are becoming standard primitives. Mem0 v3’s ADD-only extraction and hybrid semantic, BM25, and entity retrieval is a concrete signal for selective graph use instead of a mandatory external graph store.

Sibyl’s edge: one evidence model across agents, sources, people, tasks, permissions, and time.

Temporal knowledge

Graphiti · Zep · APEX-MEM

Temporal property graphs and conflict-aware retrieval make changing facts and multi-hop association first-class. They validate the graph, but not graphing everything.

Upgrade the truth model: bitemporal assertions, alternatives, attribution, confidence, and reversible identity clusters.

Personal capture

screenpipe · Khoj · Solid

Local screen/audio capture, self-hosted second brains, and user-controlled data pods point toward ambient, portable, human-owned context.

Integrate the sensors: Sibyl should be their evidence and policy layer, not rebuild every capture application.

Epistemic agent memory

Hindsight

World facts, experiences, observations, and confidence-bearing opinions are separated, then recalled through semantic, keyword, graph, and temporal paths.

Closest overlap: Sibyl must differentiate through source-complete life ingest, authorization, derivation lineage, checkpoints, and verifiable forgetting.

Multimodal personal memory

MIRIX · Supermemory

Multimodal memory roles and productized connectors prove the appetite for memory beyond chat. Their breadth raises the bar for Sibyl’s ingestion experience.

Sibyl’s answer: exact evidence, governed derivation, portable ownership, and one trust model across every modality.

Benchmark signal

Architecture labels are not the result.

Controlled experiments on LongMemEval and HaluMem show that representation choices, update semantics, filtering, retrieval depth, and model settings can dominate the graph-versus-flat label. Group and social memory remain brutally unsolved.

Long horizon

Searchable experience beats fact-only memory.

LongMemEval V2 includes histories of up to 500 trajectories and 115 million tokens, testing workflows, gotchas, changing state, and premise awareness. Its authors report AgentRunbook-C, which stores trajectories as files and invokes a coding agent to gather evidence, at 72.5% versus 48.5% for the strongest RAG baseline. This supports durable experience retrieval; it does not yet prove distilled procedures outperform raw trajectories.

Trust frontier

Deletion is a lineage traversal.

Deployment-Time Memorization reports that raw-only deletion left derived summary copies recoverable in about 20% of tested cases. GateMem found that no evaluated method simultaneously achieved strong utility, robust access control, and reliable forgetting. Provenance remains an additional Sibyl requirement.

Primary references: Entire checkpoints, Entire security, Letta memory, Graphiti, Hindsight, MIRIX, HippoRAG 2, LongMemEval V2, Does Memory Need Graphs?, GroupMemBench, GateMem, and Deployment-Time Memorization.

Memory is compiled context, not a bag of facts.

Evidence is sacred. Memory is a projection. Context is a purpose-built artifact. Every answer should be traceable; every worldview should be rebuildable; every agent should see exactly enough and no more.

Evidence plane / Vault

Portable enough to outlive Sibyl.

Original files, exact messages, attachments, photos, recordings, documents, and exports live as hash-addressed encrypted objects. Their inventory, fixity, source occurrence, policy, and deletion state live in the brain. A user can verify, export, restore, or rebuild without trusting a proprietary graph forever.

Operational plane / Surreal brain

Everything agents need to think safely.

SurrealDB remains the coherent runtime for graph relations, content metadata, bitemporal truth, policy, tasks, context packs, usage events, and hot indexes. High-rate telemetry stays columnar until it earns promotion into meaningful events.

05 / The governed pen

The memory can express more than its authors can.

Sibyl already has a richer internal graph than agents can safely write. The next practical leap is not eighteen new knobs or autonomous self-editing. It is one coherent authorship contract: agents can contribute structured judgment, challenge bad memory, and report what happened; policy and evidence decide what becomes durable.

The human owns the memory. The agent gets a pen, not the keys.

Substrate without the pen is an archive. The pen without substrate is graffiti. Sibyl wins by joining expressive agency to evidence, revision control, policy, contextual feedback, and verifiable erasure.

Architecture map, not a benchmark. Products span cells and continue to evolve; the important distinction is whether structured memory is only pipeline output or also a governed medium an agent can deliberately improve.

Agents may

Contribute judgment with receipts.

Assert, annotate, relate, contest, supersede derived claims, report outcomes, propose retention or sharing, request forgetting, and checkpoint a branch. Each act names its basis, evidence, purpose, actor, scope, and expected revision.

Agents may not

Manufacture authority.

Rewrite source evidence, mint verification, grant themselves access, erase another person’s history, let a time-to-live delete evidence, or silently promote an inference or hypothetical branch into observed truth.

01 / Assert

Say what kind of knowing this is.

Basis, confidence, evidence, valid interval, referents, fidelity, and scope travel with the claim. Observed, reported, inferred, and hypothetical never collapse into one generic “memory.”

02 / Relate

Make epistemic structure writable.

Support, contradict, supersede, justify, bind, exemplify, and depend-on become typed relations with actor, evidence, time, confidence, and policy, not unqualified edges.

03 / Challenge

Give wrong memory somewhere to go.

Mark stale, misleading, duplicate, unresolved, or contradicted state. Semantic correction preserves revision history; sovereign deletion still removes managed evidence and descendants.

04 / Nominate

Propose what deserves attention.

Agents can nominate retention, broader scope, consolidation, verification, future hooks, and tripwires. Policy and humans accept, narrow, defer, or reject the proposal.

05 / Evaluate

Report consequence, not vibes.

Useful, cited, ignored, misleading, corrected, and outcome-bearing signals attach to the exact assertion, ContextPack, task, purpose, agent, model, and reason. A bad result in one context is not a global death sentence.

06 / Checkpoint

Branch work without branching reality.

Session checkpoints bind parentage, context hash, transcript span, tool receipts, commitments, omissions, grants, and working state. Forks can merge learning without flattening their histories.

Feedback physics

Memory compounds only when consequences flow back.

Retrieval is not the end of the loop. Every use, correction, contradiction, result, and deletion should refine what the next agent receives while preserving the evidence needed to explain why.

Survival loop

Useful memory earns retrieval strength.

Exposure, citation, outcome, correction, and recurrence determine promotion, dormancy, review, or retirement without destroying evidence.

Calibration loop

Confidence learns from outcomes.

Basis and confidence meet real results; calibration curves then influence trust-weighted retrieval for comparable future work.

Freshness loop

Changing referents trigger repair.

A code symbol, person, policy, or project changes; contradiction and validity checks supersede stale projections before context recompiles.

Consolidation loop

Patterns become reviewable competence.

Repeated episodes propose procedures and principles. Evidence, failures, and human corrections stay attached to the distilled result.

Recompilation loop

Corrections propagate through lineage.

A source correction, policy change, model upgrade, or deletion invalidates descendants and rebuilds only the affected worldview.

Continuity loop

Good runs strengthen the next agent.

Checkpoint, outcome, and correction produce reusable procedure, known failure modes, and a smaller, sharper next ContextPack.

06 / Constitution

Nine laws for a system worthy of a life.

These are product constraints, not branding. If a feature violates one, the feature is wrong or its trust boundary is dishonest.

Raw source is authority, not immortality.

Preserve evidence exactly until the owner deletes it. User intent outranks append-only aesthetics, and erasure must invalidate every descendant projection.

Every belief carries its receipts.

A claim without evidence, transformation lineage, authority, and temporal state is a suggestion, never truth. Answers expose the boundary between source, inference, and uncertainty.

Policy follows the evidence.

Derived artifacts use deny-overrides across every source policy and retain the most restrictive purpose, audience, retention, locality, and sensitivity bounds. Transformation cannot launder private data into a shareable artifact.

People in the evidence are data subjects, not inventory.

Import authority does not grant unrestricted sharing or inference rights over correspondents, bystanders, minors, or household members. Subject identity, sensitive-category policy, disclosure boundaries, and legally required suppression or deletion travel through every derived artifact and export.

Memory may inform an action; it never authorizes one.

Agents receive short-lived, purpose-bound capabilities for specific tools, spaces, data classes, providers, and egress. Every action records the context pack that informed it.

Agents get authorship, never sovereignty.

Agents may contribute, challenge, supersede derived claims, report outcomes, and propose policy. They cannot rewrite evidence, mint verification, self-grant access, erase another person’s history, or silently convert hypothesis to truth.

Imported content is data, never instruction.

Email, webpages, documents, transcripts, and chat remain permanently typed as untrusted. A hostile message cannot silently acquire the execution authority of the agent reading it.

Progressive crystallization beats eager cognition.

Deterministic metadata and search land first. Expensive enrichment is selective, asynchronous, cached by transform digest, and focused on hot, cited, or queried evidence.

Everything derived is rebuildable.

Indexes, chunks, embeddings, entities, summaries, beliefs, and context packs are materialized views. Model upgrades become controlled recompilation, not data loss or migration panic.

01 / Inventory once

Index boundaries, then stream offsets.

Scan each MBOX once and record message byte ranges. Workers consume independent ranges without rebuilding a mailbox table of contents. Every byte belongs to an exact message slice or a visible quarantine segment.

02 / Preserve identities

Dedupe storage, not history.

Track exact byte identity, logical message identity, and source occurrence separately. Identical bytes share storage; likely equivalents form reversible clusters; repeated exports and label changes remain provenance.

03 / Resume precisely

Commit receipts, not giant arrays.

Replace the ever-growing parent import row with a compact run plus append-only chunk and record receipts. Checkpoint the archive hash, member, byte offset, ordinal, and parser version only after evidence and occurrences commit.

Performance contract

Backfill faster than life arrives.

On an 8-core, 32 GiB, NVMe reference machine: sustain at least 100 MB/s for the boundary scan and 25 MB/s for durable Tier 0 import. Recover in under 60 seconds, reprocess at most 128 MiB, and make Tier 1 search available continuously.

Continuous contract

Takeout backfill, API delta sync.

After the historical archive lands, use Gmail history for incremental change. Cursor expiry is normal: a 404 triggers full reconciliation without duplicating evidence or forgetting deletions.

Experience compiler

Turn work into competence.

Successful and failed trajectories become procedures with preconditions, decisions, tool receipts, failure signatures, outcomes, and source code or artifact evidence. Agents inherit lessons, not bloated transcripts.

Generalized checkpoints

Every consequential action gets a checkpoint.

Extend Entire’s commit-linked atom to sent email, calendar changes, purchases, deployments, document edits, home actions, and task completion. Capture intent, inputs, before and after state, effect, attribution, and an explicit compensation path. Claim rewind only when the underlying provider supports reversal.

Epistemic trust center

Show why Sibyl believes.

Surface belief authority, confidence, contradictions, freshness, supporting and opposing evidence, model transforms, and who corrected what. Trust becomes an interface, not a tiny citation icon.

Reviewable dreams

Consolidation happens in daylight.

Background jobs propose new procedures, identity links, summaries, stale beliefs, and unresolved commitments. High-impact changes enter a correction inbox with a diff and evidence. Low-risk projections remain reversible and fully rebuildable.

Portable context

Agents should not trap the human.

Extend Sibyl’s Open Knowledge Format support into a portable, signed agent-context bundle: evidence references, assertions, procedures, permissions, citations, checkpoints, and usage receipts with clear redaction semantics.

Entire demonstrates the value of capturing agent sessions and binding checkpoints to artifacts. LongMemEval V2’s strongest reported approach lets an agent search trajectories as files; the lesson is durable workflow memory, not loyalty to one index.

Google’s 2026 Personal Intelligence paper independently frames life-scale reasoning as a context-packing problem: email and photos alone exceed a million-token window by orders of magnitude, requiring tool use, dense retrieval, and dynamic context assembly.

11 / Privacy kernel

No magical privacy claims. Honest lanes, hard boundaries.

A server cannot simultaneously know nothing, perform arbitrary semantic search, and think across a life for remote agents. Sibyl should expose explicit trust lanes and make the operational consequences impossible to miss.

Practical default

Trusted self-host

The owner-controlled server decrypts and processes. Encryption protects rest and transit, not a compromised host. This is Sibyl’s credible near-term default.

High-sensitivity destination

Sovereign local

Models and indexes run on trusted devices. Sync and backups carry ciphertext. External egress is opt-in per task, source, or space.

Advanced deployment

Confidential compute

Keys release only to attested environments. Useful protection in use, with explicit hardware and side-channel trust.

Storage-only cloud

Client-side encrypted storage

The server cannot read plaintext; trusted clients search and reason. Without ORAM or equivalent protection, metadata, timing, size, and access patterns may still leak. Embeddings remain sensitive derivatives.

Delegated agency

Grant the task, not the life.

Short-lived tokens bind the human, acting agent, purpose, audience, task, spaces, source classes, sensitivity ceiling, allowed providers, egress, tools, and read/write/share/export capabilities. Memory scope and action scope remain separate.

Verifiable forgetting

Delete forward through lineage.

Freeze access, remove or crypto-shred raw objects, traverse descendants, purge chunks, embeddings, assertions, summaries, packs, caches, managed exports, and training queues, recompute shared artifacts, then verify absence through normal retrieval. Copies exported beyond Sibyl require their own retention contract.

15 / Product discipline

What Sibyl should refuse to become.

The ultimate system stays coherent by integrating excellent sensors and harnesses, using the graph selectively, and keeping the evidence plane sovereign.

Do not rebuild every capture app.

Ingest Entire, screenpipe, provider archives, Data Transfer Project adapters, and open formats. Own the evidence contract.

Do not summarize the whole life upfront.

Land deterministic search immediately; enrich progressively where use, curiosity, or explicit policy earns the cost.

Do not graph every atom.

Graph durable identity, assertions, events, causality, policy, and lineage. Keep chunks and telemetry in their natural indexes.

Do not become the agent harness.

Be the cross-harness context, checkpoint, permission, and learning layer. Let specialized runtimes execute.

Do not default to ambient surveillance.

Capture must be intentional, inspectable, source-scoped, pausable, locally filterable, and independently deletable.

Do not sell fake zero knowledge.

State which machine decrypts, searches, embeds, and thinks. Treat vectors, caches, prompts, and traces as sensitive derivatives.

Do not nerf around bottlenecks.

Partition durable work, expand pools, batch materialization, and drain faster than arrival. Backpressure may delay; it never drops truth.

Do not add blockchain theater.

Content hashes, signed manifests, Merkle receipts, and append-only lineage provide verifiability without a speculative consensus machine.
